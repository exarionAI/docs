# Product Family

The Sound Tracing product family is organized around four pillars: **Core engines**, **SDK bindings**, **studio tooling**, and **analysis tools**.

## Core

Native C/C++ acoustic simulation engines.

- [STCore](../core/stcore.md) — first-generation core, including an FPGA acceleration interface, maintenance mode
- [STCoreV2](../core/stcorev2.md) — second-generation core, modular path modules and lock-free data flow
- [STCore vs STCoreV2 Comparison](../core/comparison.md)

## SDK

Binding lineup that makes the Core engine usable from each platform and language.

- [SDK Overview](../sdk/overview.md)
- Planned bindings: Web, Python, Unity, Unreal Engine

## ExaStudio

A desktop application for visually authoring and debugging acoustic scenes.

- [ExaStudio Overview](../exastudio/overview.md)

## ExaTools

Utility tools for IR analysis, data conversion, batch validation, and related workflows.

- [ExaTools Overview](../exatools/overview.md)
